Screen Orientation API

🔒 锁定屏幕方向

// 锁定为竖屏
await screen.orientation.lock('portrait-primary');
// 锁定为横屏
await screen.orientation.lock('landscape-primary');

🔓 解锁屏幕方向

screen.orientation.unlock();

📱 获取当前方向

// 获取方向类型
const type = screen.orientation.type;
// 获取角度
const angle = screen.orientation.angle;

👂 监听方向变化

screen.orientation.addEventListener('change', () => {
console.log('方向已改变:', screen.orientation.type);
});
📋 支持的方向值
  • • portrait-primary - 主竖屏
  • • portrait-secondary - 倒置竖屏
  • • landscape-primary - 主横屏
  • • landscape-secondary - 倒置横屏

Screen Wake Lock API

🔋 获取屏幕唤醒锁

// 请求屏幕常亮
const wakeLock = await navigator.wakeLock.request('screen');

🔓 释放唤醒锁

wakeLock.release();

👂 监听释放事件

wakeLock.addEventListener('release', () => {
console.log('屏幕唤醒锁已释放');
});

🔍 检查API支持

if ('wakeLock' in navigator) {
// 支持 Wake Lock API
console.log('支持屏幕常亮功能');
}

🔄 完整示例

let wakeLock = null;
try {
wakeLock = await navigator.wakeLock.request('screen');
console.log('屏幕常亮已启用');
} catch (err) {
console.error('启用失败:', err.message);
}
⚠️ 重要提示
  • • 需要在安全上下文(HTTPS)中使用
  • • 页面不可见时会自动释放
  • • 需要用户激活状态才能获取
  • • 移动设备支持更好

使用场景与最佳实践

🎯 Screen Orientation API 使用场景

  • 游戏应用: 锁定横屏获得更好的游戏体验
  • 视频播放: 全屏视频时锁定横屏
  • 相机应用: 拍照时锁定特定方向
  • 数据展示: 图表展示时锁定最佳方向

💡 Screen Wake Lock API 使用场景

  • 阅读应用: 长时间阅读时保持屏幕常亮
  • 导航应用: 导航过程中防止屏幕休眠
  • 健身应用: 运动计时时保持屏幕可见
  • 演示应用: 展示内容时防止屏幕关闭
🏆 开发最佳实践
Screen Orientation API:
  • • 在用户交互后立即调用
  • • 优先尝试进入全屏模式
  • • 提供降级方案和错误处理
  • • 适时解锁方向恢复用户控制
Screen Wake Lock API:
  • • 页面不可见时自动释放
  • • 监听释放事件进行状态更新
  • • 提供用户手动控制选项
  • • 考虑电池消耗影响
